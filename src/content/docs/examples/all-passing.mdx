## What to Notice

<Aside type="note" title="Verification links: [[ file:test ]]">
The `[[ tests/test_calculator.py:test_add ]]` syntax at the end of each requirement line links that requirement to a specific test. During `sodide verify`, sodide runs each linked test and marks the requirement `✓` only if all linked tests pass.
</Aside>

<Aside type="note" title="Module scope">
The `modules: [src/]` frontmatter field declares that all implementation code for this spec lives under `src/`. sodide traces git changes and flags any modifications outside that boundary as scope violations. With 0 violations, everything stayed in scope.
</Aside>

<Aside type="note" title="All-green output">
When every requirement has at least one passing linked test, the summary line reads `N verified, 0 failing, 0 no evidence`. This is the target state — the spec is fully satisfied and every claim is backed by evidence.
</Aside>
